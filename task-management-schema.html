Smart Task & Activity Management System

Full-Stack App — Node.js + React.js + SQLite + Socket.io

Primary Key
Foreign Key
Hierarchy: Organization → Users → Teams → Projects → Tasks
Permission Model: super_admin > admin > user (org-level) + owner/admin/member (team-level)
Real-time: Socket.io broadcasts task changes to all team members instantly

✅ Implemented Features

JWT Auth + Refresh TokenAccess token (15min) + Refresh token (7d, rotated)
Role-Based Access Controlsuper_admin, admin, user + team roles
Task CRUDCreate, read, update, delete with full field support
Filtering & SortingFilter by status, priority, assignee, project
Server-side Paginationpage + limit params, returns total & totalPages
Activity LogTracks every task change with user + timestamp
Real-time UpdatesSocket.io — task changes broadcast to team room
Rate LimitingGlobal: 500/15min, Auth: 10/15min per IP
Team InvitesEmail-based invite with accept/reject flow
Kanban BoardDrag & drop tasks between status columns

🔗 Relationships

users.organization_id → organizations.id — User belongs to org (CASCADE DELETE)
refresh_tokens.user_id → users.id — Token belongs to user (CASCADE DELETE)
teams.organization_id → organizations.id — Team belongs to org
team_members.team_id + user_id → teams + users — Composite PK, many-to-many
team_invites.team_id → teams.id — Invite for a team
projects.team_id → teams.id — Project belongs to team
tasks.team_id → teams.id — Task always belongs to team
tasks.project_id → projects.id — NULL = adhoc task (not in any project)
tasks.assigned_to → users.id — SET NULL on user delete
task_activity_logs.task_id → tasks.id — Log entry for a task (CASCADE DELETE)
